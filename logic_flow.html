Step 3 — How allocation is calculated
Allocation Weight per Stock weight = quality_score × sip_modifier
SIP VERDICT MODIFIER VISUAL MEANING
🟢 ACCUMULATE NOW ×1.4
Boosted — you get 40% more allocation than standard
✅ GOOD ENTRY ×1.15
Slight boost — price is still attractive
🟡 INVEST NORMALLY ×1.0
Standard allocation — no adjustment
🟠 INVEST LESS ×0.55
Reduced — invest ~half, save rest for next dip
🔴 WAIT THIS MONTH ×0
Zero — amount redistributed to stocks at dips
HOW THE MATH WORKS
For each eligible stock: weight = quality_score × modifier
Allocation = monthly_investment × (weight / sum_of_all_weights)

So if TCS has quality=65 and ACCUMULATE (×1.4), it gets 65 × 1.4 = 91 weight.
If INFY has quality=60 and WAIT (×0), it gets 0 — and TCS gets that extra share.

Old v3 vs New v4
❌ Old Logic (v3)
→ Only asked "is this a good stock?"
→ Recommended BUY when stock was already rallying
→ You bought at high prices
→ Dips were ignored because next month = "BEARISH"
→ Allocation was based purely on quality score
→ Same stocks got same allocation regardless of price
✅ New Logic (v4)
→ Asks two questions: quality AND price level
→ SIP Entry Score is HIGH when price is LOW
→ Stocks at dips get boosted allocation (×1.4)
→ Stocks near highs get zero or reduced allocation
→ Deferred money is redirected to better-priced stocks
→ Monthly SIP discipline preserved — just smarter deployment
